# ION Network Profile ‚Äî v2.0.0 (profile version 0.5.2-draft)

This directory holds the ION network's overlays on Beckn Protocol v2.0.0. Two authored files and one generated file.

## `ion.yaml` ‚Äî Layer 2: Network policy profile

The policy overlay on Beckn. Declares rules that apply to every ION participant regardless of sector:

- Network ID, registry URL, catalog service URL
- Localization defaults (IDR currency, Asia/Jakarta timezone, id language)
- Compliance requirements (NPWP mandatory, data residency Indonesia; PPN rate sourced from current DJP/PMK regulation ‚Äî not hardcoded)
- Security requirements (Ed25519 signing, BLAKE2b-512 digest)
- Mandatory action sets for BAP and BPP, split by Beckn-native vs ION-extension
- Settlement model and allowed payment rails
- Schema pack conformance matrix
- Active sectors (trade, logistics) and reserved sector slots
- Experimental `x-ion-context-extensions` (spine/pattern/branches on the wire ‚Äî optional in v0.5.2, targeted mandatory in v0.6)

This file does **not** replace `beckn.yaml`. Beckn's core transport specification lives at:
<https://github.com/beckn/protocol-specifications-v2>

## `ion-endpoints.yaml` ‚Äî Layer 3: Protocol endpoint extensions

The endpoints ION adds to Beckn. Specifies 8 ION protocol extension endpoints that are NOT in Beckn 2.0 core:

- `/raise`, `/on_raise`, `/raise_status`, `/on_raise_status`, `/raise_details`, `/on_raise_details` ‚Äî network dispute adjudication
- `/reconcile`, `/on_reconcile` ‚Äî settlement reconciliation

These use the same Context schema, signing scheme, and ACK/NACK patterns as Beckn 2.0 core ‚Äî they are additive endpoints, not a protocol fork. Referenced from `ion.yaml` under `x-ion-actions.ionExtensions`.

## `ion-with-beckn.yaml` ‚Äî Composed spec (generated)

Beckn v2.0.0 core + ION profile + ION extension endpoints + all ION attribute packs, composed into a single validation target. Generated by `tools/generate_composed.py`. Do not edit directly.

## Conformance

All ION implementations must satisfy:

1. **Beckn 2.0.0 core** (see `beckn.yaml` upstream) ‚Äî the 30 Beckn-native endpoints and their schemas
2. **`ion.yaml`** ‚Äî the ION network policy overlay (Layer 2)
3. **`ion-endpoints.yaml`** ‚Äî the 8 ION protocol extension endpoints (Layer 3)
4. **The ION core attribute packs** (`schema/extensions/core/*`) for every sector (Layer 4)
5. **The relevant ION sector attribute packs** (`schema/extensions/trade/*` or `schema/extensions/logistics/*`) for the sectors they operate in (Layer 5)

See `docs/ION_Layer_Model.md` for the full 5-layer explanation.
See `docs/ION_Beckn_Conformance.md` for the full conformance checklist.
